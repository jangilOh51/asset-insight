## 1. 제품 개요

### 비전
증권사 앱을 여러 개 열지 않아도, 한 화면에서 전체 자산 현황을 파악하고 AI 인사이트로 투자 판단을 지원받는 개인 투자 통합 플랫폼.

### 핵심 문제
- 국내·해외 증권 계좌가 여러 증권사에 분산되어 있어 전체 자산을 한눈에 파악하기 어렵다
- 일별 자산 변화 추이를 별도로 기록하지 않으면 히스토리를 볼 수 없다
- 종목·세금·전략 분석에 AI를 활용하고 싶지만 도구가 없다

### 현재 범위
- **사용자**: 단독 사용 (1인)
- **계좌**: 한국투자증권 + 키움증권
- **환경**: 로컬 Docker Compose

## 3. 기능 요구사항

### 3.1 핵심 기능 (Phase 1 — 완료)

#### F-01 통합 포트폴리오 조회
- 모든 활성 계좌의 잔고를 병렬 조회해 단일 화면에 표시
- 계좌별 + 전체 합산: 평가금액, 투자원금, 평가손익, 수익률, 예수금
- 종목 테이블: 종목명·현재가·수량·평균단가·매입금액·평가금액·평가손익·수익률·당일등락·비중
- 컬럼 헤더 클릭 정렬 (오름/내림 토글)
- 60초 자동 갱신, 수동 새로고침

#### F-02 일별 자산 스냅샷
- 평일 18:00 KST 자동 저장 (APScheduler)
- 실시간 조회 시 BackgroundTask로 저장 (10분 쓰로틀)
- 저장 항목: 종목별 포지션 + 계좌별 일 요약

#### F-03 자산 트렌드 분석
- 기간: 일별·주별·월별 전환
- 차트: 총자산 추이 + 기간별 수익률 표 (1주/1달/3달/1년)
- 벤치마크 비교: KOSPI·S&P500·NASDAQ 오버레이

#### F-04 계좌 관리
- 증권사 계좌 등록·수정·삭제·활성화 토글
- 계좌별 API 키/시크릿 저장 (DB 암호화 예정)
- 표시 순서 조정 (위/아래)
- 계좌 연동 테스트 (verify)

---

### 3.2 시황·자동화 (Phase 2 — 완료)

#### F-05 시황 페이지
- 지수 카드: KOSPI·KOSDAQ·S&P500·NASDAQ·USD/KRW
- 보유 종목 트리맵 (비중별 면적, 수익률별 색상)
- 데이터 5분 캐시 (Redis)

#### F-06 투자 목표 설정 & 진행률
- 목표 이름·금액 설정 (활성 목표 최대 1개)
- 대시보드 홈 진행률 카드 (프로그레스바 + %)
- 목표 달성/교체/삭제

---

### 3.3 AI 인사이트 (Phase 3 — 완료)

#### F-07 AI 월간 리포트
- 현재 포트폴리오 기반 Claude API 생성
- 섹션: 포트폴리오 현황 / 종목별 분석 / 리스크 요인 / 다음 달 전략
- Redis 캐시 1시간

#### F-08 AI 투자 전략서
- 리스크 성향 (보수·중립·공격), 투자 기간 (1·3·5·10년) 선택
- 7섹션 전략 문서 생성
- Redis 캐시 2시간 (동일 파라미터+포트폴리오 구성)

#### F-09 세금 시뮬레이터
- 매도 시뮬레이션: 국내주식·해외주식·국내ETF·해외ETF·채권
- 증권거래세·양도소득세·배당소득세 룰 기반 계산
- 연간 기본공제 250만원 적용

#### F-10 절세 캘린더
- 연간 8개 법정 일정 (ISA 만기, 연금계좌 납입, 양도소득세 신고 등)
- D-day 표시, 월별 필터

---

### 3.4 확장 기능 (Phase 4 — 진행 중)

#### F-11 종목 상세 AI 리포트 (완료)
- 보유 종목 행에서 AI 분석 버튼 클릭
- 섹션: 종목 개요 / 보유 현황 분석 / 리스크 요인 / 매매 관점
- Redis 캐시 30분 (현재가 100원 단위 버킷)
- 미보유 종목 요청 → 404

#### F-12 인앱 알림 (완료)
- 트리거: 목표 달성, 목표 50·75·90% 마일스톤, 이상 수익(+20%), 이상 손실(-15%)
- Redis 24시간 중복 방지
- 사이드바 벨 아이콘 + 미읽음 배지
- 드롭다운 패널: 읽음 처리, 전체 삭제

#### F-13 수동 자산 관리 (완료)
- 부동산·예금·기타 자산 CRUD
- 포트폴리오 총자산에 합산

## 4. 비기능 요구사항

### 4.1 성능
| 항목 | 목표 |
|------|------|
| 포트폴리오 실시간 조회 응답 | < 3초 (병렬 API 호출) |
| AI 리포트 생성 (캐시 미스) | < 15초 |
| 페이지 초기 로드 | < 2초 |
| 차트 렌더링 | 클라이언트 dynamic import, SSR 비활성화 |

### 4.2 신뢰성
- 증권사 API 장애 시 → HTTP 502/503 반환 (데이터 손실 없음)
- Redis 장애 시 → 캐시 미스로 동작, 쓰로틀 우회
- 스케줄러 작업 실패 → 로그 기록, 다음 주기에 재시도

### 4.3 보안 (현재 단독 사용 기준)
- API 키·시크릿 → DB 저장 (암호화 미적용, 향후 개선 TD)
- CORS: `localhost:3000`만 허용
- `.env` → `.gitignore` 처리

### 4.4 테스트
- 모든 API 엔드포인트 단위 테스트 필수
- 외부 의존성(DB·Redis·증권사 API·Claude API) 전부 Mock
- 현재 156개 테스트, 0 failed
