# 수동 자산 관리 — 화면 요구사항

## 목적
증권 API로 연결되지 않는 자산(부동산, 예금/적금, 가상화폐, 비상장주식 등)을
직접 등록·수정·삭제하여 포트폴리오 총 자산에 포함시킨다.

---

## 기능 요구사항

### FR-01 자산 목록
- 등록된 수동 자산 카드 목록 표시
- 카드: 아이콘(이모지) / 자산명 / 자산 유형 배지 / 현재 가치 / 수익(구매가 대비)
- 자산 없으면 빈 상태 + "자산 추가하기" CTA

### FR-02 자산 유형
| 유형 코드 | 표시명 | 기본 이모지 |
|-----------|--------|-------------|
| `real_estate` | 부동산 | 🏠 |
| `deposit` | 예금/적금 | 🏦 |
| `crypto` | 가상화폐 | ₿ |
| `private_equity` | 비상장주식 | 📈 |
| `pension` | 연금/퇴직금 | 🏛️ |
| `other` | 기타 | 💼 |

### FR-03 자산 추가 모달
- 필드:
  - 자산 유형 (버튼 선택)
  - 자산명 (텍스트, 필수)
  - 현재 가치 (KRW, 필수)
  - 매입 가격 (KRW, 선택 — 수익률 계산용)
  - 메모 (텍스트, 선택, 최대 200자)
- 저장 성공 시 모달 닫기 + 목록 갱신

### FR-04 자산 수정 모달
- FR-03과 동일 필드 (자산 유형 변경 가능)

### FR-05 자산 삭제
- 삭제 confirm: "자산을 삭제할까요?"

### FR-06 활성/비활성 토글
- 비활성 자산은 포트폴리오 합산에서 제외

### FR-07 통계 요약
- 상단: 수동 자산 총액 / 자산 수 / 총 수익(구매가 대비)

### FR-08 포트폴리오 통합
- 포트폴리오 홈(`/`) 총 자산에 활성 수동 자산 합산
- 포트폴리오 홈 계좌 목록에 수동 자산 카드 표시 (별도 섹션)

---

## 비기능 요구사항
- 최대 너비: 672px
- 통화는 KRW 고정 (향후 USD 지원 예정)

---

## API 의존성

| API | 용도 |
|-----|------|
| `GET /api/v1/assets` | 수동 자산 목록 |
| `POST /api/v1/assets` | 자산 추가 |
| `PATCH /api/v1/assets/{id}` | 자산 수정 |
| `DELETE /api/v1/assets/{id}` | 자산 삭제 |
| `PATCH /api/v1/assets/{id}/toggle` | 활성/비활성 |
