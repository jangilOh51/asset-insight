# 사용 가이드 — 화면 요구사항

## 목적
서비스 기능에 대한 설명과 사용법을 정적 문서 형태로 제공한다.
목차로 섹션 간 빠른 이동이 가능하다.

---

## 기능 요구사항

### FR-01 목차 사이드바
- 좌측 고정 사이드바 (200px), sticky top
- 섹션 목록: 서비스 개요 / 포트폴리오(홈) / 포트폴리오 상세 / 트렌드 분석 / 자산 히스토리 / 계좌 관리 / FAQ
- 클릭 시 해당 섹션으로 smooth scroll
- 현재 활성 섹션 하이라이트 (cyan 색상)

### FR-02 본문 섹션
각 섹션은 다음으로 구성:
- `Heading`: 섹션 제목 (id 속성 포함, scrollMarginTop: 80px)
- `Sub`: 소제목
- `P`: 본문 텍스트
- `Table`: key-value 2열 표
- `InfoBox`: 파란 배경 안내 박스 (팁)
- `WarnBox`: 노란 배경 경고 박스 (주의사항)
- `Badge`: 인라인 태그

### FR-03 섹션 목록 및 주요 내용
| 섹션 | ID | 주요 내용 |
|------|----|-----------|
| 서비스 개요 | overview | 플랫폼 소개, 색상 규칙, 지원 기능 표 |
| 포트폴리오 (홈) | dashboard | 화면 구성, 자동 갱신, DB 저장 안내 |
| 포트폴리오 상세 | portfolio | 좌우 패널 설명, 바차트 스냅샷 안내 |
| 트렌드 분석 | trend | 기간 탭, 벤치마크 비교, 수익률 표 설명 |
| 자산 히스토리 | history | 스냅샷 조회, 날짜 클릭 드릴다운 설명 |
| 계좌 관리 | accounts | 계좌 등록, API 키 발급, 검증 방법 |
| FAQ | faq | 자주 묻는 질문 5개 이상 |

### FR-04 FAQ 항목 (최소 5개)
1. 데이터가 없다고 나와요 → 매일 18:00 스냅샷 저장 설명
2. 환경변수 계좌가 계좌 목록에 없어요 → env-kis 자동 표시 설명
3. 수익이 빨강인 이유 → 한국 증권 관례 설명
4. API 키 어디서 발급하나요 → KIS/키움 발급처 안내
5. 금액이 이상하게 표시돼요 → fmt() 형식 설명 (₩86,360,000)

## 비기능 요구사항

- 정적 페이지 (API 호출 없음)

- 본문은 overflowY: auto로 독립 스크롤
