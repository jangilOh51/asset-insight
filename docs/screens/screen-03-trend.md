# 트렌드 분석 — 화면 요구사항

## 목적
DB에 저장된 스냅샷 데이터를 기반으로 기간별 수익률 추이를 차트와 표로 분석한다.
KOSPI / S&P500 / NASDAQ 벤치마크와 비교할 수 있다.

---

## 기능 요구사항

### FR-01 계좌 선택
- 활성 계좌 2개 이상이면 드롭다운으로 선택
- 1개이면 계좌명 + 증권사 텍스트로 표시 (드롭다운 없음)
- 계좌 없으면 빈 상태 안내 + "계좌 추가하기 →" CTA

### FR-02 기간 탭
- 일간(90일) / 주간(52주) / 월간(24개월) 탭 전환
- 탭 선택에 따라 API limit 파라미터 변경
- 활성 탭: `#06B6D4` 배경

### FR-03 벤치마크 토글
- "지수 비교" 버튼으로 KOSPI/S&P500/NASDAQ 오버레이 ON/OFF
- 활성 상태: `#22D3EE` 색상 + 반투명 배경

### FR-04 수익률 추이 라인 차트
- X축: 날짜, Y축: 수익률(%) — 첫 데이터 대비 누적 수익률
- 내 포트폴리오 선: `#06B6D4` 실선
- KOSPI: `#F59E0B` 점선, S&P500: `#10B981` 점선, NASDAQ: `#8B5CF6` 점선
- 기준선(0%): 회색 점선
- Tooltip: 날짜 + 각 계열 수익률
- 범례: 하단 중앙 (벤치마크 표시 시에만)
- SSR false (dynamic import 필수)

### FR-05 기간별 수익률 비교 표
- 행: 내 포트폴리오 / KOSPI / S&P500 / NASDAQ (벤치마크 ON 시)
- 열: 1주일 / 1개월 / 3개월 / 1년
- 데이터 없으면 "—" 표시
- 수익/손실 색상 일관 적용

### FR-06 최근 데이터 포인트 표
- 최근 10개 데이터 포인트 (날짜 / 총 자산 / 평가손익 / 수익률)
- 최신순 정렬

### FR-07 로딩 상태
- 차트 영역: 단일 스켈레톤 블록 (height: 240px)
- 표 영역: 별도 스켈레톤

### FR-08 에러 상태
- 빨간 배너: "트렌드 데이터를 불러오지 못했습니다."

### FR-09 빈 데이터 상태
- 차트 영역 중앙: "데이터를 수집 중입니다 / 매일 18:00 스냅샷이 저장됩니다"

## 비기능 요구사항

- 최대 너비: 900px
- 벤치마크 데이터 출처: Yahoo Finance (httpx 직접 호출, 1시간 Redis 캐시)

---

## API 의존성

| API | 용도 |
|-----|------|
| `GET /api/v1/trend/{account_id}?period={daily\|weekly\|monthly}&limit={n}` | 트렌드 데이터 포인트 |
| `GET /api/v1/trend/benchmark/returns?from={date}` | KOSPI/S&P500/NASDAQ 수익률 |
| `GET /api/v1/accounts` | 계좌 목록 |
