# 자산 히스토리 — 화면 요구사항

## 목적
DB에 저장된 일별 스냅샷 데이터를 날짜 단위로 조회한다.
날짜를 클릭하면 해당 날짜의 종목 내역을 드릴다운으로 확인한다.

---

## 기능 요구사항

### FR-01 계좌 선택 드롭다운
- 활성 계좌 목록으로 드롭다운 표시
- 계좌 변경 시 날짜 선택 초기화 (selectedDate = '')
- 계좌 없으면 빈 상태 + "계좌 추가하기 →" CTA

### FR-02 저장 데이터 수 표시
- 드롭다운 옆: "N일 데이터 저장됨" 또는 "저장된 데이터 없음"

### FR-03 일별 자산 요약 테이블
- 정렬: 최신순 (내림차순)
- 컬럼: 날짜 / 총 자산 / 매입금액 / 평가금액 / 평가손익 / 수익률 / 예수금 / 종목수
- 클릭 가능한 날짜 행 (cursor: pointer)
- 선택된 행: `rgba(6,182,212,0.08)` 배경 + 날짜에 "선택" 배지
- hover 시 `rgba(255,255,255,0.02)` 배경

### FR-04 날짜 드릴다운 — 종목 내역 테이블
- 날짜 행 클릭 시 해당 날짜 종목 목록 로드 (테이블 하단에 표시)
- 컬럼: 종목명 / 심볼 / 시장 / 수량 / 매입금액 / 평가금액 / 평가손익 / 수익률
- 시장 배지: KR = `#EF4444` 빨강, US = `#60A5FA` 파랑
- 종목 없으면: "해당 날짜 종목 데이터가 없습니다"

### FR-05 스냅샷 없는 경우
- 일별 테이블 영역에 안내 문구:
  "저장된 스냅샷이 없습니다"
  "매일 18:00 자동 저장 또는 [수동 저장] 링크"
- 수동 저장 링크: `GET /api/v1/snapshot/run`

### FR-06 로딩 상태
- 일별 요약 테이블: 5행 스켈레톤 (height: 36px)
- 종목 내역 테이블: 3행 스켈레톤

## 비기능 요구사항

- 최대 너비: 1100px
- 테이블 가로 스크롤 (`overflowX: auto`)
- 숫자 폰트: JetBrains Mono

---

## API 의존성

| API | 용도 |
|-----|------|
| `GET /api/v1/accounts` | 활성 계좌 목록 |
| `GET /api/v1/snapshot/summary/{account_no}?limit=365` | 일별 자산 요약 (최근 365일) |
| `GET /api/v1/snapshot/positions/{account_no}?date={YYYY-MM-DD}` | 선택 날짜 종목 내역 |
