# 포트폴리오 홈 — 화면 요구사항

## 목적
모든 활성 계좌의 자산을 통합 조회하는 메인 대시보드.
실시간 잔고 기반으로 총 자산·손익을 한눈에 파악한다.

---

## 기능 요구사항

### FR-01 총 자산 표시
- 모든 활성 계좌의 평가금액 + 예수금 합산값을 Hero 영역에 표시
- 금액 형식: `fmt()` 사용 (₩86,360,000 형식, 만/억 단위 금지)
- 총 자산이 0이면 "—" 표시

### FR-02 평가손익 표시
- Hero 하단에 손익 금액 + 수익률(%) 함께 표시
- 수익: `#EF4444` (빨강), 손실: `#60A5FA` (파랑) — 한국 관례
- 수익이면 "+" 부호 앞에 붙임

### FR-03 StatCard — 예수금 / 평가손익
- 예수금: 금액 + "총 자산의 X.X%" 힌트 텍스트
- 평가손익: 금액 + 수익률 서브텍스트
- 2열 그리드 레이아웃

### FR-04 종목 현황 트리맵
- 보유 종목을 평가금액 비중에 따라 트리맵으로 시각화
- 색상 = **당일 등락률** (`day_change_pct`) 기준 (상승: #EF4444 계열, 보합: #525252, 하락: #22C55E 계열)
  - +3% 이상: `#EF4444`, +2%: `#DC2626`, +1%: `#B91C1C`, +0.1%: `#7F1D1D`
  - ±0.1% 이내: `#525252` (보합)
  - -1%: `#166534`, -2%: `#15803D`, -3%: `#16A34A`, -3% 초과 하락: `#22C55E`
- 셀 내부에 종목명 + 당일 등락률(%) 표시
- 범례(상승/보합/하락) 상단 우측에 표시
- SSR false (dynamic import 필수)

### FR-05 계좌 목록 카드
- 활성 계좌 목록을 카드 형태로 나열
- 카드 클릭 → `/portfolio/{account_id}` 이동
- 계좌 없으면 빈 상태 + "계좌 추가하기 →" CTA

### FR-06 자동 갱신
- useSWR refreshInterval: 60,000ms (60초)
- 조회 시 백그라운드로 DB에 자동 저장 (BackgroundTask)
- TopBar 우상단 "새로고침" 버튼 (수동 갱신)
- TopBar에 마지막 업데이트 시각 표시

### FR-07 로딩 상태
- 스켈레톤 UI: Hero 영역, StatCard, 계좌 목록에 각각 표시
- `@keyframes pulse` 애니메이션 적용

### FR-08 에러 상태
- API 오류 시 상단 빨간 배너 표시
- 메시지: "실시간 데이터를 불러오지 못했습니다. API 키 및 계좌 설정을 확인하세요."
