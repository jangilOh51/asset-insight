## 4. Phase별 로드맵

### Phase 1 — 핵심 포트폴리오 (진행 중)
> 목표: KIS 실시간 데이터 기반 완전한 포트폴리오 트래킹

```
[완료] 프로젝트 구조 · Docker · DB 모델
[완료] KIS API 연동 (실시간 잔고)
[완료] 포트폴리오 상세 화면
[진행] 대시보드 홈 실데이터 연결       ← P1-01
[진행] 계좌관리 디자인 정비            ← P1-02
[진행] AssetBarHistory 실데이터        ← P1-03
[예정] 트렌드 분석 화면               ← P1-04
[예정] 벤치마크 비교                  ← P1-05, P1-06
```

### Phase 2 — 시황 & 자동화
> 목표: 시장 데이터 자동 수집 + 대시보드 완성

```
[예정] 시황 페이지 (KOSPI/나스닥/환율)  ← P2-01
[예정] 섹터 트리맵 실데이터            ← P2-02
[예정] 목표 설정 & 진행률              ← P2-04
```

### Phase 3 — AI 인사이트
> 목표: Claude API 연동 자동 리포트 + 전략서

```
[예정] AI 월간 포트폴리오 리포트       ← P3-01
[예정] 전략서 생성 페이지             ← P3-02
[예정] 세금 시뮬레이터                ← P3-03
```

### Phase 4 — 확장
> 목표: 멀티 계좌 + 알림 + 모바일 최적화

```
[예정] 미래에셋 등 멀티 증권사        ← P4-01
[예정] 푸시 알림                     ← P4-03
```

상세 계획서는 `docs/plans/` 폴더에 작업 시작 전 작성한다.

```
docs/plans/
├── S1-01-dashboard-realdata.md    ← 대시보드 홈 실데이터
├── S1-02-bar-history-api.md       ← 바차트 스냅샷 연결

```

### 계획서 표준 양식

```markdown
# [ID] [작업명] 계획서

## 목적 / 배경
```
