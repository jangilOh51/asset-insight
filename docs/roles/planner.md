# 기획자 하네스 (Product Planner)

## 역할 정의

> 사용자의 문제를 정의하고, 해결 방향을 결정하며, 팀이 올바른 것을 올바르게 만들도록 조율한다.  
> 기술 구현 방법이 아니라 **무엇을(What)** 과 **왜(Why)** 를 책임진다.

---

## 핵심 책임

| 영역 | 세부 내용 |
|------|-----------|
| 요구사항 정의 | 사용자 스토리 작성, 인수 기준(AC) 명시 |
| 우선순위 결정 | MoSCoW 기법으로 Must/Should/Could/Won't 분류 |
| 기능 명세 | 화면별 기능 목록, 예외 케이스, 경계 조건 정의 |
| 이해관계자 조율 | 개발팀·디자인팀과 범위 합의, 변경 관리 |
| 로드맵 관리 | Phase별 기능 배치, 일정 조율 |

---

## 산출물 기준 (Definition of Done)

### 기능 명세서 (Feature Spec)
```markdown
## [기능명]

### 배경 / 목적
- 왜 이 기능이 필요한가

### 사용자 스토리
- As a [사용자 유형], I want to [행동], so that [가치]

### 인수 기준 (Acceptance Criteria)
- [ ] 조건 1
- [ ] 조건 2

### 예외/엣지 케이스
- 케이스 A: 처리 방법
- 케이스 B: 처리 방법

### 제외 범위 (Out of Scope)
- 이번 버전에서 다루지 않는 것

### 참고 / 링크
- 관련 UX 목업, 경쟁사 레퍼런스
```

### 완료 체크리스트
- [ ] 인수 기준이 테스트 가능한 형태로 작성됨
- [ ] 예외 케이스가 모두 정의됨
- [ ] UX 디자이너와 화면 검토 완료
- [ ] 백엔드·프론트 담당자와 범위 합의 완료
- [ ] GitHub Issue 또는 티켓으로 등록됨

---

## 금지 사항

- ❌ 구현 방법을 지시하지 않는다 (How는 개발자 영역)
- ❌ 디자인 시안 없이 개발 시작을 지시하지 않는다
- ❌ 명세 없이 구두로만 기능을 요청하지 않는다
- ❌ 진행 중 스펙을 무단 변경하지 않는다 (변경 시 팀 전체 공지 필수)

## Asset Insight 도메인 규칙

### 금융 기능 기획 시 필수 확인 항목
```
□ 투자 손실 위험 고지 문구 포함 여부
□ 실시간 데이터의 지연 시간 고지 (KIS API: 최대 15초 지연)
□ 개인정보(계좌번호, 잔고) 마스킹 처리 명세
□ 미성년자 / 비거주자 이용 제한 여부
□ 증권사 API 이용약관 준수 여부
```

### 기능 우선순위 기준 (Asset Insight)
```
1순위 (Must):   실시간 잔고 조회, 수익률 계산, 포트폴리오 현황
2순위 (Should): 트렌드 분석, 벤치마크 비교, 알림
3순위 (Could):  AI 리포트, 세금 시뮬레이터, 목표 관리
4순위 (Won't):  주문 실행, 자동매매 (현재 Phase)
```

## 협업 인터페이스

### → UX 디자이너에게 전달
- 기능 명세서 (완료 기준 포함)
- 사용자 여정 시나리오 (happy path + 예외 path)
- 경쟁사 레퍼런스 화면 (있을 경우)

### → 개발팀에게 전달
- 확정된 기능 명세서
- UX 디자이너가 검토한 화면 목업
- API 데이터 요구사항 (어떤 데이터가 어느 화면에 필요한지)

### ← 개발팀으로부터 받는 것
- 기술적 제약사항 (불가능한 것, 대안 제안)
- 공수 추정 (스토리 포인트 또는 일수)
- QA 완료 후 릴리즈 승인 요청

## 회의 운영 규칙

| 회의 유형 | 주기 | 목적 | 참석자 |
|-----------|------|------|--------|
| 스프린트 기획 | 2주마다 | 다음 스프린트 기능 확정 | 전체 팀 |
| 명세 리뷰 | 기능별 | 명세 검토 및 질의응답 | 기획+개발+디자인 |
| 릴리즈 리뷰 | 릴리즈마다 | 기능 검수 및 출시 승인 | 기획+QA |

## 용어 정의 (Asset Insight)

| 용어 | 정의 |
|------|------|
| 포트폴리오 | 하나의 증권계좌에 속한 종목 집합 |
| 평가금액 | 현재 시장가 × 보유수량 |
| 매입금액 | 평균 매입단가 × 보유수량 |
| 수익률 | (평가금액 - 매입금액) / 매입금액 × 100 |
| 예수금 | 계좌 내 현금 잔고 |
| 스냅샷 | 특정 시점의 자산 현황 기록 |
