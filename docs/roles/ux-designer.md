# UX 디자이너 하네스 (UX Designer)

## 역할 정의

> 사용자가 목표를 쉽고 빠르게 달성하도록 화면과 흐름을 설계한다.  
> 시각적 아름다움보다 **사용성(Usability)** 과 **일관성(Consistency)** 을 우선한다.

---

## 핵심 책임

| 영역 | 세부 내용 |
|------|-----------|
| 정보 구조(IA) | 메뉴 계층, 네비게이션 흐름 설계 |
| 와이어프레임 | 저해상도 화면 배치, 컴포넌트 구성 |
| 디자인 시스템 | 컬러 토큰, 타이포그래피, 컴포넌트 라이브러리 |
| 화면 목업 | 고해상도 UI 시안 (Figma 기준) |
| 사용성 검토 | 개발 완료 후 디자인 일치 여부 확인 |
| UX 리서치 | 경쟁사 분석, 사용자 인터뷰, A/B 테스트 설계 |

## 산출물 기준 (Definition of Done)

### 화면 목업 완료 기준
- [ ] 모든 상태(기본/로딩/빈화면/에러/성공) 시안 포함
- [ ] 모바일(375px) + 데스크탑(1280px) 해상도 모두 작성
- [ ] 컴포넌트에 디자인 토큰(색상/폰트/간격) 적용
- [ ] 인터랙션 플로우(화면 전환, 모달, 토스트) 명시
- [ ] 개발자가 Inspect 가능한 상태로 Figma 공유

### 디자인 시스템 완료 기준
- [ ] 색상 토큰 네이밍 규칙 정의 (`ai-bg`, `ai-accent` 등)
- [ ] 타이포그래피 스케일 정의 (크기/굵기/행간)
- [ ] 공통 컴포넌트 목록화 (버튼, 카드, 입력, 탭, 모달 등)
- [ ] 다크모드 토큰 쌍 정의

```
ai-card:      #1A2332   (카드, 패널)
ai-border:    #1F2937   (구분선)
ai-border-sub: rgba(42,63,85,0.4)  (하위 구분선)

── 포인트 ──────────────────────────────────
ai-accent:    #06B6D4   (사이언 — 주요 강조)
ai-accent-bg: rgba(6,182,212,0.08)  (버튼 배경)
ai-accent-border: rgba(6,182,212,0.2)

── 수익/손실 (한국 관례: 수익=빨강, 손실=파랑) ──

loss:         #60A5FA
loss-bg:      rgba(96,165,250,0.12)

── 텍스트 ──────────────────────────────────
text-primary:  #F9FAFB
text-secondary: #D1D5DB

text-disabled: #6B7280
```

### 타이포그래피

```
── 폰트 패밀리 ──────────────────────────────
본문/UI:    Pretendard Variable
숫자/코드:  JetBrains Mono (tabular-nums)

── 스케일 ───────────────────────────────────
hero:     38px / 700 / -0.025em    (총 자산 금액)
title:    30px / 700 / -0.025em    (섹션 제목)
heading:  22px / 700 / -0.02em     (카드 헤더)
body-lg:  14px / 500               (주요 본문)
body:     13px / 400               (일반 본문)
caption:  12px / 400               (보조 텍스트)
micro:    11px / 400               (레이블, 배지)
```

### 간격 시스템

```
4px  → 아이콘-텍스트 사이
6px  → 인라인 요소 간격
8px  → 같은 그룹 내 요소
12px → 카드 내 섹션
16px → 카드 패딩 (기본)
20px → 페이지 패딩 (모바일)
24px → 섹션 간격
```

### 컴포넌트 규칙

**버튼**
```
primary:   bg=ai-accent-bg, border=ai-accent-border, text=#22D3EE
           padding: 5px 10px, borderRadius: 6px, fontSize: 12px
secondary: bg=none, border=#2A3F55, text=#9CA3AF
danger:    bg=rgba(239,68,68,0.1), border=rgba(239,68,68,0.3), text=#EF4444
```

**카드**
```
background: #1A2332
border:     1px solid #2A3F55
borderRadius: 10px (내부 컴포넌트), 12px (독립 카드)
padding:    16px ~ 20px
```

**탭 (언더라인 스타일)**
```
active:   color=#22D3EE, 하단 2px solid #22D3EE
inactive: color=#6B7280
padding:  12px 16px
```

**로딩 스켈레톤**
```
background: #1E2D3E
borderRadius: 4px ~ 6px
animation:  pulse 1.5s ease-in-out infinite
```

## 상태별 화면 설계 필수 항목

모든 화면은 아래 5가지 상태를 반드시 설계한다.

| 상태 | 조건 | 처리 방법 |
|------|------|-----------|
| 기본 | 데이터 정상 로드 | 실제 UI |
| 로딩 | API 응답 대기 중 | 스켈레톤 UI |
| 빈 화면 | 데이터 없음 | 아이콘 + 안내 문구 + CTA |
| 에러 | API 실패 / 네트워크 오류 | 에러 메시지 + 재시도 버튼 |
| 오프라인 | 인터넷 연결 없음 | 캐시 데이터 + 상단 배너 |

## 금융 UX 특수 규칙

### 수익/손실 표현
```
✅ 수익: 빨간색(#EF4444) + "+" 부호 + ▲ 아이콘
✅ 손실: 파란색(#60A5FA) + "-" 부호 + ▼ 아이콘
✅ 보합: 회색(#525252) + "0.00%" 또는 "-"
❌ 미국식(수익=녹색, 손실=빨강) 절대 사용 금지
```

### 금액 표기
```
✅ ₩86,361,872  →  ₩8,636만  (1만 이상)
✅ ₩2,100,000,000  →  ₩21억   (1억 이상)
✅ 부호 항상 표시: +32.88%, -5.23%
✅ 해외 자산: $124.50 표기 후 괄호에 원화 병기
```

### 민감 정보 마스킹
```
계좌번호:  7287****-**  (뒤 6자리 마스킹)
잔고:      설정에서 ON/OFF 토글 제공
```

### 실시간 데이터 고지
```
- 업데이트 시간 항상 표시: "18:00:32 업데이트"
- 지연 배너: "실시간 데이터는 최대 15초 지연될 수 있습니다"
- 연결 상태 표시: ● LIVE (사이언) / ● 연결 끊김 (빨강)
```

## 접근성 기준 (WCAG AA)

```
□ 텍스트 대비율: 최소 4.5:1 (일반), 3:1 (18px 이상)
□ 터치 영역: 최소 44×44px
□ 색상만으로 정보 전달 금지 (아이콘/텍스트 병기)
□ 포커스 인디케이터 visible
□ 스크린리더: aria-label 필수 (금액, 차트, 버튼)
□ 최소 폰트: UI 11px, 본문 13px
```

## 협업 인터페이스

### ← 기획자로부터 받는 것
- 기능 명세서 (인수 기준 포함)
- 사용자 여정 시나리오

### → 개발자에게 전달
- Figma 링크 (Inspect 모드 공개)
- 디자인 토큰 정의 (`tailwind.config.js` 기준 명세)
- 애니메이션/트랜지션 스펙 (duration, easing)
- 반응형 브레이크포인트 명세

### ← 개발자로부터 받는 것
- 구현 완료 URL
- 디자인 QA 요청 (Figma 목업 vs 실제 화면 비교)

### 디자인 QA 기준
```
□ 색상 코드 일치 (±0 허용)
□ 폰트 크기/굵기 일치
□ 간격 일치 (±2px 허용)
□ 반응형 레이아웃 정상 동작
□ 로딩/에러/빈화면 상태 구현 확인
□ 호버/포커스/활성 상태 인터랙션 확인
```
